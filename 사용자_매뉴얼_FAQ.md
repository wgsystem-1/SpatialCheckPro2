# Spatial Check Pro FAQ (자주 묻는 질문)

---

## 일반 질문

### Q1. 프로그램을 처음 사용하는데 어떻게 시작하나요?

**A**: [빠른_시작_가이드.md](빠른_시작_가이드.md)를 참고하세요. 5분 안에 검수를 시작할 수 있습니다.

**간단한 순서**:
1. 프로그램 실행
2. 폴더 선택
3. 검수 설정
4. 검수 실행
5. 결과 확인

---

### Q2. 어떤 파일 형식을 검수할 수 있나요?

**A**: 현재는 **File Geodatabase (.gdb)** 형식만 지원합니다.

- ✅ File Geodatabase (.gdb)
- ❌ Shapefile (.shp) - 향후 지원 예정
- ❌ GeoPackage (.gpkg) - 향후 지원 예정

---

### Q3. 검수에 얼마나 시간이 걸리나요?

**A**: 데이터 크기와 검수 단계에 따라 다릅니다.

| 데이터 크기 | 검수 단계 | 예상 시간 |
|------------|----------|----------|
| 소규모 (< 10만 피처) | 전체 (1~5단계) | 5~10분 |
| 중규모 (10~50만 피처) | 전체 (1~5단계) | 15~30분 |
| 대규모 (50만~100만 피처) | 전체 (1~5단계) | 30~60분 |
| 초대규모 (> 100만 피처) | 전체 (1~5단계) | 1시간 이상 |

**팁**: 특정 단계만 선택하여 검수하면 시간을 단축할 수 있습니다.

---

### Q4. 검수 중에 프로그램을 종료해도 되나요?

**A**: 검수 중 프로그램을 종료하면 진행 중인 검수가 중단됩니다. 검수 결과는 중단 시점까지의 결과만 저장됩니다.

**권장 사항**:
- 검수가 완료될 때까지 대기
- 급하게 종료해야 하는 경우 "취소" 버튼 사용

---

## 검수 관련 질문

### Q5. 모든 단계를 검수해야 하나요?

**A**: 아니요. 필요한 단계만 선택하여 검수할 수 있습니다.

**예시**:
- 지오메트리 오류만 확인하고 싶다면 → 3단계만 선택
- 속성값 오류만 확인하고 싶다면 → 4단계만 선택

---

### Q6. 검수 규칙을 수정할 수 있나요?

**A**: 네, `Config/` 폴더의 CSV 파일을 수정하면 됩니다.

**규칙 파일 위치**:
- `Config/1_table_check.csv`: 테이블 검수 규칙
- `Config/2_schema_check.csv`: 스키마 검수 규칙
- `Config/3_geometry_check.csv`: 지오메트리 검수 규칙
- `Config/4_attribute_check.csv`: 속성 관계 검수 규칙
- `Config/5_relation_check.csv`: 공간 관계 검수 규칙

**주의사항**:
- CSV 파일 수정 후 프로그램 재시작 권장
- 규칙 파일 형식이 올바른지 확인 필요

---

### Q7. 검수 결과는 어디에 저장되나요?

**A**: 검수 결과는 다음 위치에 저장됩니다.

1. **데이터베이스**: `ValidationResults.db` (자동 저장)
2. **로그 파일**: `Logs/app-YYYY-MM-DD.log`
3. **내보낸 파일**: 사용자가 선택한 위치 (Shapefile, 보고서)

---

### Q8. 오류 위치를 지도에서 확인하고 싶어요.

**A**: 오류를 Shapefile로 내보낸 후 GIS 소프트웨어에서 확인할 수 있습니다.

**방법**:
1. 검수 결과 화면에서 "내보내기" 버튼 클릭
2. Shapefile 형식 선택
3. 저장 위치 선택
4. QGIS 또는 ArcGIS에서 원본 FileGDB와 오류 Shapefile 동시 로드
5. 오류 위치 확인

---

## 성능 관련 질문

### Q9. 검수가 너무 느려요. 속도를 높일 수 있나요?

**A**: 다음 방법을 시도해보세요.

1. **병렬 처리 활성화**
   - 설정에서 병렬 처리 옵션 활성화
   - CPU 코어 수에 따라 성능 향상

2. **불필요한 검수 규칙 비활성화**
   - 필요한 단계만 선택
   - 각 단계에서 불필요한 규칙 비활성화

3. **스트리밍 모드 사용**
   - 대용량 파일 처리 시 자동 활성화
   - 메모리 사용량 감소로 성능 향상

4. **시스템 성능 확인**
   - 다른 프로그램 종료
   - 디스크 공간 확인
   - 메모리 사용량 확인

---

### Q10. 메모리 부족 오류가 발생해요.

**A**: 다음 방법을 시도해보세요.

1. **스트리밍 모드 활성화**
   - 설정에서 스트리밍 모드 활성화
   - 메모리 사용량 감소

2. **검수 단계 분리**
   - 전체 검수를 한 번에 실행하지 말고 단계별로 나누어 실행
   - 예: 1~2단계 → 3단계 → 4~5단계

3. **시스템 메모리 확보**
   - 다른 프로그램 종료
   - 시스템 메모리 업그레이드 고려

---

## 오류 관련 질문

### Q11. "파일을 열 수 없습니다" 오류가 발생해요.

**A**: 다음을 확인하세요.

1. **파일 경로 확인**
   - 파일 경로에 한글이나 특수문자가 없는지 확인
   - 경로가 너무 긴지 확인 (Windows 최대 260자)

2. **파일 접근 권한 확인**
   - 파일이 다른 프로그램에서 열려있는지 확인
   - 파일 읽기 권한 확인

3. **파일 형식 확인**
   - FileGDB 형식이 올바른지 확인
   - 파일이 손상되지 않았는지 확인

---

### Q12. 검수 결과에 오류가 너무 많아요. 어떻게 해야 하나요?

**A**: 다음 순서로 처리하세요.

1. **심각도별 분류**
   - CRITICAL 오류부터 우선 처리
   - MAJOR → MINOR → INFO 순서로 처리

2. **오류 유형별 집계**
   - 가장 많이 발생한 오류 유형부터 처리
   - 통계 차트에서 확인 가능

3. **오류 위치 확인**
   - Shapefile로 내보내기
   - GIS 소프트웨어에서 오류 위치 확인
   - 원본 데이터와 비교하여 수정

---

### Q13. 검수 결과를 다른 컴퓨터로 옮기고 싶어요.

**A**: 다음 파일을 복사하세요.

1. **데이터베이스 파일**: `ValidationResults.db`
2. **로그 파일**: `Logs/` 폴더 (선택사항)
3. **내보낸 파일**: Shapefile, 보고서 등

**주의사항**:
- 다른 컴퓨터에도 동일한 프로그램이 설치되어 있어야 함
- 데이터베이스 파일 경로가 동일해야 함

## 설정 관련 질문

### Q14. 기본 폴더 경로를 변경하고 싶어요.

**A**: 설정 화면에서 변경할 수 있습니다.

1. 사이드바에서 "설정" 클릭
2. "기본 폴더 경로" 수정
3. "저장" 버튼 클릭

---

### Q15. 검수 규칙 파일 경로를 변경할 수 있나요?

**A**: 설정 화면에서 변경할 수 있습니다.

1. 사이드바에서 "설정" 클릭
2. "검수 규칙 파일 경로" 수정
3. "저장" 버튼 클릭

**기본 경로**: `Config/` 폴더

## 기타 질문

### Q16. 프로그램을 업데이트하면 기존 검수 결과가 사라지나요?

**A**: 아니요. 검수 결과는 데이터베이스 파일(`ValidationResults.db`)에 저장되므로, 이 파일을 보관하면 기존 결과를 유지할 수 있습니다.

**권장 사항**:
- 업데이트 전 데이터베이스 파일 백업
- 중요한 검수 결과는 보고서로 저장

---

### Q17. 여러 FileGDB를 한 번에 검수할 수 있나요?

**A**: 현재는 한 번에 하나의 FileGDB만 검수할 수 있습니다. 여러 FileGDB를 검수하려면 각각 개별적으로 실행해야 합니다.

**향후 계획**: 배치 검수 기능 추가 예정

---

### Q18. 검수 결과를 Excel로 내보낼 수 있나요?

**A**: 현재는 Shapefile과 PDF/HTML 보고서만 지원합니다. Excel 내보내기는 향후 추가 예정입니다.

**대안**:
- HTML 보고서를 Excel로 변환 가능
- 데이터베이스 파일을 직접 열어 Excel로 내보내기 가능

---

### Q19. 프로그램 사용 중 문제가 발생했어요. 어떻게 해야 하나요?

**A**: 다음 순서로 확인하세요.

1. **로그 파일 확인**
   - `Logs/app-YYYY-MM-DD.log` 파일 확인
   - 오류 메시지 및 스택 트레이스 확인

2. **문제 해결 섹션 참조**
   - 사용자 매뉴얼의 "문제 해결" 섹션 참조

3. **기술 지원 문의**
   - 이메일: support@example.com
   - 전화: 02-XXXX-XXXX
   - 로그 파일과 함께 문의하면 더 빠른 해결 가능

---

### Q20. ISO 19157 표준을 준수하나요?

**A**: 네, ISO 19157 데이터 품질 표준을 준수합니다.

**준수 항목**:
- ✅ 완전성 (Completeness)
- ✅ 논리적 일관성 (Logical Consistency)
- ✅ 위치 정확도 (Positional Accuracy)
- ✅ 시간 정확도 (Temporal Accuracy)
- ✅ 주제 정확도 (Thematic Accuracy)
