# Spatial Check Pro 아이콘 및 로고 추가 가이드

WPF 애플리케이션에 아이콘, 로고, 회사 로고를 추가하는 방법을 안내합니다.

---

## 1. 아이콘 및 로고 파일 준비

### 1.1 필요한 파일

| 파일 유형 | 파일명 예시 | 용도 | 권장 크기 |
|----------|------------|------|----------|
| **애플리케이션 아이콘** | `app.ico` | 윈도우 타이틀바, 실행 파일 아이콘 | 16x16, 32x32, 48x48, 256x256 포함 |
| **애플리케이션 로고** | `logo.png` | UI 내부 로고 표시 | 200x200px (투명 배경) |
| **회사 로고** | `company_logo.png` | 회사 정보 표시 | 150x50px (가로형) |

### 1.2 파일 형식 요구사항

- **아이콘 (.ico)**: Windows 아이콘 형식, 여러 크기 포함
- **로고 이미지 (.png)**: 투명 배경 지원, PNG 형식 권장
- **색상 모드**: RGB 또는 RGBA (투명도 지원)

## 2. 파일 구조 생성

### 2.1 폴더 구조

프로젝트에 다음 폴더 구조를 생성합니다:

```
SpatialCheckPro.GUI/
├── Assets/
│   ├── Icons/
│   │   └── app.ico          (애플리케이션 아이콘)
│   └── Images/
│       ├── logo.png         (애플리케이션 로고)
│       └── company_logo.png (회사 로고)
```

### 2.2 폴더 생성 방법

1. Visual Studio에서 프로젝트 우클릭
2. "추가" → "새 폴더" 선택
3. `Assets` 폴더 생성
4. `Assets` 폴더 내에 `Icons`, `Images` 폴더 생성

## 3. 애플리케이션 아이콘 추가

### 3.1 .ico 파일 준비

**아이콘 파일 요구사항**:
- 형식: `.ico` (Windows 아이콘)
- 포함 크기: 16x16, 32x32, 48x48, 256x256
- 도구: 온라인 변환기 또는 IcoFX 같은 전문 도구 사용

**온라인 변환기 예시**:
- https://convertio.co/kr/png-ico/
- https://www.icoconverter.com/

### 3.2 프로젝트에 아이콘 추가

1. **파일 추가**
   - `Assets/Icons/` 폴더에 `app.ico` 파일 복사
   - Visual Studio에서 프로젝트 우클릭 → "추가" → "기존 항목"
   - `app.ico` 파일 선택

2. **프로젝트 파일 수정**

`SpatialCheckPro.GUI.csproj` 파일에 다음을 추가:

```xml
<PropertyGroup>
  <!-- 기존 속성들... -->
  
  <!-- 애플리케이션 아이콘 설정 -->
  <ApplicationIcon>Assets\Icons\app.ico</ApplicationIcon>
</PropertyGroup>

<ItemGroup>
  <!-- 아이콘 파일을 리소스로 포함 -->
  <None Include="Assets\Icons\app.ico">
    <CopyToOutputDirectory>PreserveNewest</CopyToOutputDirectory>
  </None>
</ItemGroup>
```

3. **App.xaml에 아이콘 설정**

`App.xaml` 파일 수정:

```xml
<Application x:Class="SpatialCheckPro.GUI.App"
             xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
             xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
             DispatcherUnhandledException="Application_DispatcherUnhandledException">
    <!-- 애플리케이션 아이콘 설정 -->
    <Application.Resources>
        <ResourceDictionary>
            <!-- 아이콘 리소스 정의 -->
            <BitmapImage x:Key="AppIcon" UriSource="pack://application:,,,/Assets/Icons/app.ico"/>
            
            <ResourceDictionary.MergedDictionaries>
                <ResourceDictionary Source="pack://application:,,,/Styles/ModernStyles.xaml"/>
                <ResourceDictionary Source="pack://application:,,,/Styles/StageResources.xaml"/>
                <ResourceDictionary Source="pack://application:,,,/Styles/Theme.Dashboard.xaml"/>
            </ResourceDictionary.MergedDictionaries>
        </ResourceDictionary>
    </Application.Resources>
</Application>
```

4. **MainWindow.xaml에 아이콘 설정**

`MainWindow.xaml` 파일의 Window 태그에 Icon 속성 추가:

```xml
<Window x:Class="SpatialCheckPro.GUI.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        xmlns:local="clr-namespace:SpatialCheckPro.GUI.Views"
        Title="Spatial Check Pro" 
        Height="1000" Width="1800"
        MinHeight="720" MinWidth="1280"
        WindowStartupLocation="CenterScreen"
        Background="White"
        Icon="pack://application:,,,/Assets/Icons/app.ico"
        Closing="MainWindow_Closing">
```

## 4. 애플리케이션 로고 추가

### 4.1 로고 이미지 준비

**로고 파일 요구사항**:
- 형식: `.png` (투명 배경 지원)
- 크기: 200x200px (정사각형) 또는 적절한 비율
- 해상도: 고해상도 (2x, 3x 버전도 준비 가능)

### 4.2 프로젝트에 로고 추가

1. **파일 추가**
   - `Assets/Images/logo.png` 파일 복사
   - Visual Studio에서 프로젝트 우클릭 → "추가" → "기존 항목"
   - `logo.png` 파일 선택

2. **프로젝트 파일 수정**

`SpatialCheckPro.GUI.csproj` 파일에 다음을 추가:

```xml
<ItemGroup>
  <!-- 이미지 파일을 리소스로 포함 -->
  <Resource Include="Assets\Images\logo.png">
    <CopyToOutputDirectory>PreserveNewest</CopyToOutputDirectory>
  </Resource>
</ItemGroup>
```

또는:

```xml
<ItemGroup>
  <None Include="Assets\Images\logo.png">
    <CopyToOutputDirectory>PreserveNewest</CopyToOutputDirectory>
  </None>
</ItemGroup>
```

3. **App.xaml에 로고 리소스 정의**

`App.xaml` 파일의 ResourceDictionary에 추가:

```xml
<Application.Resources>
    <ResourceDictionary>
        <!-- 로고 이미지 리소스 정의 -->
        <BitmapImage x:Key="AppLogo" UriSource="pack://application:,,,/Assets/Images/logo.png"/>
        
        <ResourceDictionary.MergedDictionaries>
            <!-- 기존 리소스들... -->
        </ResourceDictionary.MergedDictionaries>
    </ResourceDictionary>
</Application.Resources>
```

4. **MainWindow.xaml에 로고 표시**

타이틀바에 로고 추가:

```xml
<!-- 타이틀바 -->
<Border Grid.Row="0" Background="#F0F0F0" BorderBrush="#E0E0E0" BorderThickness="0,0,0,1" Grid.ColumnSpan="3">
    <Grid>
        <Grid.ColumnDefinitions>
            <ColumnDefinition Width="Auto"/>  <!-- 로고 영역 추가 -->
            <ColumnDefinition Width="*"/>
            <ColumnDefinition Width="Auto"/>
        </Grid.ColumnDefinitions>

        <!-- 로고 -->
        <Image Grid.Column="0" 
               Source="{StaticResource AppLogo}" 
               Width="32" Height="32" 
               Margin="16,0,12,0"
               VerticalAlignment="Center"/>

        <!-- 타이틀 -->
        <StackPanel Grid.Column="1" Orientation="Horizontal" 
                   VerticalAlignment="Center" Margin="0,0,16,0">
            <TextBlock Text="국가기본도 DB 검수 프로그램" 
                      VerticalAlignment="Center" 
                      FontWeight="SemiBold" 
                      FontSize="16"/>
            <Border Background="#3B82F6" 
                   CornerRadius="10" 
                   Padding="6,2" 
                   Margin="8,0,0,0">
                <TextBlock x:Name="VersionText" Text="v2.1.0" 
                          Foreground="White" 
                          FontSize="12" 
                          FontWeight="SemiBold"/>
            </Border>
        </StackPanel>
    </Grid>
</Border>
```

## 7. 빌드 및 테스트

### 7.1 빌드 전 확인 사항

- [ ] 모든 이미지 파일이 올바른 위치에 있는지 확인
- [ ] 프로젝트 파일에 파일 참조가 추가되었는지 확인
- [ ] XAML에서 리소스 경로가 올바른지 확인

### 7.2 빌드 및 확인

1. **프로젝트 빌드**
   ```bash
   dotnet build SpatialCheckPro.GUI.csproj -c Release
   ```

2. **확인 사항**
   - 실행 파일 아이콘 확인 (파일 탐색기에서)
   - 프로그램 실행 시 윈도우 타이틀바 아이콘 확인
   - UI 내부 로고 표시 확인

3. **배포 시 확인**
   - `publish` 폴더에 이미지 파일이 복사되었는지 확인
   - 배포된 실행 파일에서 아이콘 정상 표시 확인

## 8. 문제 해결

### 8.1 아이콘이 표시되지 않음

**원인**: 파일 경로 오류 또는 빌드 설정 문제

**해결 방법**:
1. 파일이 프로젝트에 포함되어 있는지 확인
2. `CopyToOutputDirectory` 설정 확인
3. 리소스 경로 확인 (`pack://application:,,,/` 형식)

### 8.2 이미지가 표시되지 않음

**원인**: 리소스 경로 오류 또는 파일 형식 문제

**해결 방법**:
1. 이미지 파일이 `Resource` 또는 `Content`로 설정되어 있는지 확인
2. XAML에서 `Source` 속성 경로 확인
3. 이미지 파일 형식 확인 (PNG 권장)

### 8.3 배포 후 이미지가 표시되지 않음

**원인**: 배포 시 이미지 파일이 복사되지 않음

**해결 방법**:
1. `CopyToOutputDirectory` 속성을 `Always` 또는 `PreserveNewest`로 설정
2. `publish` 폴더에 이미지 파일 존재 확인
3. 상대 경로 대신 `pack://application:,,,/` URI 사용

## 9. 권장 사항

### 9.1 이미지 최적화

- **파일 크기**: 가능한 한 작게 유지 (로딩 속도 향상)
- **해상도**: 필요한 크기의 2배 정도 (고해상도 디스플레이 대응)
- **형식**: PNG (투명도 필요 시), JPG (사진 이미지)

### 9.2 아이콘 디자인

- **일관성**: 애플리케이션 로고와 아이콘이 일관된 디자인
- **가독성**: 작은 크기에서도 명확하게 보이도록
- **브랜딩**: 회사 브랜드 가이드라인 준수

### 9.3 성능 고려

- **리소스 로딩**: 이미지는 리소스로 포함하여 빠른 로딩
- **캐싱**: 자주 사용하는 이미지는 리소스로 정의하여 재사용
- **지연 로딩**: 필요할 때만 로드 (대용량 이미지의 경우)
