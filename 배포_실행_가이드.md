# 배포 스크립트 실행 가이드

## 📍 실행 위치

배포 스크립트는 **프로젝트 루트 디렉토리**에서 실행해야 합니다.

```
G:\SpatialCheckPro\          ← 여기서 실행
├── 배포_스크립트.ps1         ← 이 파일
├── SpatialCheckPro.sln
├── SpatialCheckPro\
└── SpatialCheckPro.GUI\
```

## 🚀 실행 방법

### 방법 1: PowerShell에서 직접 실행 (권장)

1. **PowerShell 열기**
   - Windows 키 + X → "Windows PowerShell" 또는 "터미널" 선택
   - 또는 시작 메뉴에서 "PowerShell" 검색

2. **프로젝트 디렉토리로 이동**
   ```powershell
   cd G:\SpatialCheckPro
   ```

3. **스크립트 실행**
   ```powershell
   .\배포_스크립트.ps1
   ```

### 방법 2: Visual Studio 터미널에서 실행

1. Visual Studio에서 프로젝트 열기
2. **보기** → **터미널** 메뉴 선택
3. 터미널에서 다음 명령 실행:
   ```powershell
   .\배포_스크립트.ps1
   ```

### 방법 3: 파일 탐색기에서 실행

1. 파일 탐색기에서 `G:\SpatialCheckPro` 폴더 열기
2. `배포_스크립트.ps1` 파일을 우클릭
3. **PowerShell로 실행** 선택

## ⚠️ 실행 정책 오류 발생 시

만약 "실행 정책으로 인해 스크립트를 실행할 수 없습니다" 오류가 발생하면:

### 해결 방법 1: 실행 정책 변경 (일시적)

```powershell
Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process
.\배포_스크립트.ps1
```

### 해결 방법 2: 스크립트 내용 직접 실행

스크립트 내용을 복사하여 PowerShell에 직접 붙여넣기

### 해결 방법 3: dotnet publish 명령 직접 실행

```powershell
dotnet publish SpatialCheckPro.GUI/SpatialCheckPro.GUI.csproj `
  -c Release `
  -r win-x64 `
  --self-contained true `
  -p:PublishSingleFile=true `
  -p:IncludeNativeLibrariesForSelfExtract=true `
  -p:PublishTrimmed=false `
  -o ./publish
```

## 📂 출력 위치

스크립트 실행 후 배포 파일은 다음 위치에 생성됩니다:

```
G:\SpatialCheckPro\
└── publish\                    ← 여기에 생성됨
    ├── SpatialCheckPro.GUI.exe
    └── Config\
        ├── 1_table_check.csv
        ├── 2_schema_check.csv
        └── ...
```

## ✅ 실행 확인

스크립트가 정상적으로 실행되면:

1. **빌드 진행 메시지** 표시
2. **"✅ 빌드 완료!"** 메시지 표시
3. **생성된 파일 정보** 표시 (파일명, 크기, 경로)
4. **Config 디렉토리 확인** 메시지 표시

## 🔧 문제 해결

### 문제: "파일을 찾을 수 없습니다"

**해결**: 프로젝트 루트 디렉토리에서 실행했는지 확인

```powershell
# 현재 위치 확인
Get-Location

# 프로젝트 루트로 이동
cd G:\SpatialCheckPro
```

### 문제: "빌드 실패"

**해결**: 
1. 실행 중인 애플리케이션 종료
2. Visual Studio 종료
3. 다시 빌드 시도

### 문제: "GDAL Native 파일 복사 실패"

**해결**: 
1. `bin\Release\net8.0-windows\` 디렉토리에서 실행 중인 프로세스 확인
2. 해당 프로세스 종료 후 재시도
